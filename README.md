## خطوات الرفع على GitHub Pages

1. أنشئ مستودعاً جديداً على GitHub (مثال: `saraya`).
2. ارفع **جميع** ملفات هذه الحزمة إلى جذر المستودع (root) — وليس داخل مجلد فرعي.
3. من المستودع: `Settings` ← `Pages`.
4. تحت `Source` اختر الفرع `main` والمجلد `/ (root)` ثم `Save`.
5. انتظر دقيقة، سيظهر الرابط:
   `https://USERNAME.github.io/saraya/`

## التشغيل والتثبيت عبر Safari (iPhone / iPad / Mac)

### على iPhone أو iPad
1. افتح الرابط في **Safari** (وليس Chrome — التثبيت كتطبيق يعمل فقط من Safari على iOS).
2. اضغط زر **المشاركة** (المربع مع السهم لأعلى).
3. مرّر واختر **«إضافة إلى الشاشة الرئيسية»** (Add to Home Screen).
4. اضغط **«إضافة»** — سيظهر أيقونة سرايا على شاشتك ويفتح كتطبيق مستقل بملء الشاشة.

### على Mac (Safari 17+)
1. افتح الرابط في Safari.
2. من شريط القوائم: `ملف` ← **«إضافة إلى Dock»** (Add to Dock).
3. سيُثبّت كتطبيق مستقل في الـ Dock.

## ملاحظات مهمة

- **HTTPS مطلوب**: الـ Service Worker و PWA يعملان فقط عبر HTTPS. GitHub Pages يوفّره تلقائياً.
- **التحديث**: عند رفع نسخة جديدة من `index.html`، غيّر رقم النسخة في أعلى `sw.js` (سطر `CACHE_VERSION`) ليُجبر المتصفح على جلب النسخة الجديدة.
- **التزامن السحابي مفعّل ويعمل من أي نطاق (Firebase Firestore)**: النظام يتزامن لحظياً بين جميع الأجهزة. كل تعديل (معاملة، مشروع، مستخدم، مؤقّت، إعداد) يُحفظ محلياً للسرعة ثم يُرفع تلقائياً إلى Firestore، ويصل فوراً لباقي الأجهزة عبر مستمعين لحظيين. مشروع Firebase: `rh-task-f5af5`. **لا حاجة لأي إعداد إضافي** — النظام لا يستخدم Firebase Authentication، لذا التزامن عبر Firestore يعمل من رابط GitHub مباشرةً كما كان يعمل سابقاً.
- **التخزين المحلي (localStorage) طبقة سرعة ودعم offline**: عند عودة الاتصال يتزامن مع السحابة. المصدر الموثوق هو Firestore.

## الأمان — قواعد Firestore

مفتاح Firebase (`apiKey`) مكتوب داخل `index.html` وهذا طبيعي لتطبيقات الويب (المفتاح ليس سرّاً ولا يمنح صلاحيات بذاته). الحماية الحقيقية تأتي من **Firestore Security Rules**:

- تأكد أن قواعد Firestore على مجموعة `rh_system` ليست مفتوحة للعموم بلا قيود.
- بما أن النظام لا يستخدم Firebase Authentication، فإن التحكم بالوصول يعتمد على إبقاء الرابط خاصاً بالفريق + ضبط القواعد. إن أردت حماية أقوى مستقبلاً، يمكن إضافة طبقة Firebase Auth (خطوة منفصلة، ليست مطلوبة الآن).
